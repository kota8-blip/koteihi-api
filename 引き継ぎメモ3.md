## 次にやること（APIサーバー構築）

### ステップ1: Expressサーバー最小構成（今日のゴール）
`c:\プログラミング\my-express-api\index.js` を作成:

```js
const express = require('express');
const app = express();

app.use(express.json());

app.get('/api/expenses', (req, res) => {
  res.json({ message: 'ok' });
});

app.listen(3001, () => {
  console.log('サーバー起動: http://localhost:3001');
});
```

起動: `node index.js`
確認: ブラウザで `http://localhost:3001/api/expenses` → `{"message":"ok"}` が返れば成功

### ステップ2: NuxtからAPIを叩く
axiosで `localhost:3001/api/expenses` にリクエストを投げてレスポンスを確認

### ステップ3: DBを繋ぐ
PostgreSQL か MySQL をインストールしてExpressからCRUD操作

### ステップ4: localStorageをAPIに置き換える
- `SAVE_TO_STORAGE` → axios.post
- `LOAD_FROM_STORAGE` → axios.get
- mutationからactionへ移行（非同期処理のため）

---

## 重要な注意点
- mutationは**同期のみ**。APIは非同期（async/await）なので、API呼び出しは必ず**action**に書く
- Nuxtは3000番ポートを使うので、Expressは**3001番**で起動する

## 次にやること（認証機能追加）

### ステップ1: DBにusersテーブルを追加 ✅
```sql
CREATE TABLE users (
  id SERIAL PRIMARY KEY,
  username VARCHAR(50) UNIQUE NOT NULL,
  password VARCHAR(255) NOT NULL
);
```

### ステップ2: Express側にユーザー登録・ログインAPIを作成 ✅
- `POST /api/register` → bcryptでパスワードをハッシュ化してDB保存
- `POST /api/login` → bcryptで検証、成功時にJWTトークン発行
- 使用パッケージ: `bcrypt`, `jsonwebtoken`
- JWT_SECRETは `.env` で管理（`expiresIn: '7d'`）

### ステップ3: 認証ミドルウェアを実装 ✅
- `authenticateToken` 関数をindex.js末尾に定義
- GET/POST/PUT/DELETE `/api/transactions` の第2引数に適用
- `/api/register` と `/api/login` はtoken不要のため適用なし
- tokenなし→401、token無効→403を返す

### ステップ4: transactionsテーブルにuser_idを追加 ✅
- pgAdminで以下のSQLを実行済み:
  ```sql
  ALTER TABLE transactions ADD COLUMN user_id INTEGER REFERENCES users(id);
  ```
- GET: `WHERE user_id = $1` で自分のデータのみ取得
- POST: `req.user.userId` をINSERTに含めてDB保存
- DELETE: `WHERE id = $1 AND user_id = $2` で自分のデータのみ削除
- PUT: `WHERE id = $5 AND user_id = $6` で自分のデータのみ更新
